# Glossário

Termos técnicos usados neste repositório, explicados em linguagem simples.

| Termo | O que significa |
|---|---|
| **ADR** (Architecture Decision Record) | Documento que registra uma decisão arquitetural importante, o contexto e as alternativas consideradas |
| **BDD** (Behaviour-Driven Development) | Técnica de desenvolvimento onde o comportamento esperado do sistema é descrito em linguagem natural antes de ser implementado |
| **CI/CD** | Integração Contínua (testar cada alteração automaticamente) e Entrega Contínua (publicar alterações automaticamente) |
| **IaC** (Infrastructure as Code) | Gerenciar servidores, redes e outros recursos de infraestrutura através de arquivos de código, em vez de configuração manual |
| **Monorrepo** | Estratégia de armazenar código de múltiplos projetos em um único repositório |
| **Nuvem** | Conjunto de servidores remotos acessados pela internet, que rodam aplicações e armazenam dados |
| **Provedor de nuvem** | Empresa que oferece infraestrutura de nuvem (ex: AWS, Google Cloud, Azure) |
| **Workload** | Conjunto de componentes de software que executam uma função específica |

## Como este glossário cresce

Sempre que um termo técnico for usado em um documento sem explicação no próprio texto, ele deve ser adicionado aqui.
